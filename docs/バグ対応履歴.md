# メンバー一覧：オーナー表示名の修正

**日付**: 2026年3月  
**対象**: `/members` ページ

## 事象
一般ユーザーでログイン時、オーナーが「オーナー」固定表示になる（オーナーログイン時は登録名で表示される）

## 原因
一般ユーザーがオーナーの `profiles` を読めない Firestore ルール

## 修正
**firestore.rules** の `profiles` read ルールを変更：
- 所属会社のオーナーのプロフィール読み取りを許可（メンバー一覧表示用）
- その他メンバーは従来どおり非公開

## デプロイ
```bash
firebase deploy --only firestore:rules
```

# 案件一覧：一般ユーザーに他担当の案件が表示される

**日付**: 2026年3月  
**対象**: `/projects`（案件一覧）ページ

## 事象
一般ユーザー（例：石井さん）でログイン時、担当が石井さんのみの案件が表示されるはずが、柴田さん担当の案件も表示されてしまう

## 原因
`app/crm/deals/page.tsx` の `visibleDeals` フィルタに「自分が作成した案件は常に表示」の例外があり、作成者であれば担当者でなくても表示されていた

## 修正
**app/crm/deals/page.tsx** の該当例外を削除：
- 一般ユーザーは担当者（assignee）に自分が含まれる案件のみ表示
- オーナーは従来どおり全案件表示

# 顧客一覧：売上列に他担当者の売上が表示される

**日付**: 2026年3月  
**対象**: `/customers`（顧客一覧）ページ

## 事象
一般ユーザー（石井さん）でログイン時、売上列に藤井さんなど他担当者の売上も表示されてしまう

## 原因
`app/crm/customers/page.tsx` の売上列で、担当者別内訳を全員分表示していた

## 修正
**app/crm/customers/page.tsx** の売上列表示を変更（表示のみ）：
- 一般ユーザーは担当者別内訳をログインユーザー分のみ表示
- オーナーは従来どおり全担当者の内訳を表示
