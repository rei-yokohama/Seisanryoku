# メンバー一覧：オーナー表示名の修正

**日付**: 2026年3月  
**対象**: `/members` ページ

## 事象
一般ユーザーでログイン時、オーナーが「オーナー」固定表示になる（オーナーログイン時は登録名で表示される）

## 原因
一般ユーザーがオーナーの `profiles` を読めない Firestore ルール

## 修正
**firestore.rules** の `profiles` read ルールを変更：
- 所属会社のオーナーのプロフィール読み取りを許可（メンバー一覧表示用）
- その他メンバーは従来どおり非公開

## デプロイ
```bash
firebase deploy --only firestore:rules
```

# 案件一覧：一般ユーザーに他担当の案件が表示される

**対象**: `/projects`（案件一覧）ページ

## 原因
`app/crm/deals/page.tsx` の `visibleDeals` フィルタに「自分が作成した案件は常に表示」の例外があり、作成者であれば担当者でなくても表示されていた

- 一般ユーザーは担当者（assignee）に自分が含まれる案件のみ表示
- オーナーは従来どおり全案件表示

# マネージャーロール追加

**日付**: 2026年3月  
**対象**: 権限設定、顧客一覧・案件一覧

## 概要
テレアポマネージャー（例：石井さん）が業務委託者の顧客・案件を管理し、アサインできるようにする機能。ロール追加は行わず、既存の権限設定で実現。

## マネージャー権限の設定方法
オーナーが 設定 > メンバー > [石井さん] > 編集 で以下を設定：

| 設定項目 | 場所 | 設定内容 |
|----------|------|----------|
| 顧客の閲覧範囲 | 「顧客権限をさらに制御する」 | 閲覧範囲: **特定の雇用形態** → **業務委託** を選択 |
| 案件の閲覧範囲 | 「案件権限をさらに制御する」 | 同上 |
| 業務委託者の作成 | 「メンバー権限をさらに制御する」 | **メンバー作成を許可** + **作成可能な雇用形態: 業務委託** |
| メニュー表示 | メンバー編集画面 | 顧客・案件・メンバーなど必要なメニューにチェック |

## 実装内容

### 担当者未設定の顧客・案件を表示
マネージャーが業務委託者をアサインするには、担当者未設定の顧客・案件を一覧で確認する必要がある。従来は担当者未設定のものは非表示だったため、以下を実装。

**修正ファイル**
- **lib/visibilityPermissions.ts**: `filterByVisibleUids` に `includeUnassigned` パラメータを追加
- **app/crm/customers/page.tsx**: 顧客権限が `viewScope: specific_employment_types` かつ `viewOthersData: true` の場合、担当者未設定も表示
- **app/crm/deals/page.tsx**: 案件権限が同条件の場合、担当者未設定も表示

**適用範囲**
顧客権限・案件権限で「閲覧範囲: 特定の雇用形態」を選択したユーザーのみ。一般ユーザー（自分のデータのみ）やオーナー（全件表示）には影響なし。

## ロール構成（現状）
- **オーナー**: 全権限
- **メンバー**: メニュー表示権限・顧客権限・案件権限・メンバー権限を個別設定
- **マネージャー**: 上記の権限設定で「特定の雇用形態（業務委託）」を選択したユーザーが該当

# ドライブ：CORS エラーでファイルアップロードできない

**日付**: 2026年3月  
**対象**: `/drive`（ドライブ）ページ

## 事象
ドライブでファイルをアップロードしようとすると CORS エラーが発生し、アップロードが開始できない

```
Access to XMLHttpRequest at 'https://firebasestorage.googleapis.com/...' from origin 'https://crm.sof10.net' has been blocked by CORS policy
```

## 原因
1. **環境変数のバケット名が誤っていた**: `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=seisanryoku.firebasestorage.app` を指定していたが、このバケットはプロジェクトに存在しない
2. **実際のバケット**: `seisanryoku.appspot.com` が正しい Storage バケット（CORS は既に設定済み）

## 修正
**`.env.local`** の `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` を変更：
- 変更前: `seisanryoku.firebasestorage.app`
- 変更後: `seisanryoku.appspot.com`

## CORS 設定に必要なツール（gsutil）

Firebase Storage の CORS は Firebase Console では設定できないため、`gsutil` コマンドを使用する。

### インストール（macOS / Homebrew）

```bash
brew install google-cloud-sdk
```

### 認証・CORS 適用手順

```bash
# 1. Google アカウントでログイン（Firebase プロジェクトのオーナー権限が必要）
gcloud auth login

# 2. プロジェクトを指定
gcloud config set project seisanryoku

# 3. CORS をバケットに適用（プロジェクトルートで cors.json がある状態で実行）
gsutil cors set cors.json gs://seisanryoku.appspot.com
```

### バケット一覧の確認

```bash
gcloud storage buckets list --project=seisanryoku
```

## 補足
- `cors.json` は既に正しく設定済み（crm.sof10.net, localhost など）
- CORS の適用は `gsutil cors set cors.json gs://seisanryoku.appspot.com` で実施済み
- 本番環境（Firebase Hosting 等）の環境変数も同様に `seisanryoku.appspot.com` に設定すること
